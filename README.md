# ウェアラブルセンサを用いた歩行解析および生体力学的パラメータ推定プロジェクト

本プロジェクトは、ウェアラブルセンサを用いて、通常は大掛かりな設備が必要な歩行動態（関節角度や床反力）を深層学習によって推定するための研究用コードベースです。
この README は、プロジェクトの目的、各モジュールの役割、および実行方法について、共同研究者や今後のプロジェクト参加者が全体構造を明確に把握できるよう体系的に整理されています。

---

## 1. Quick Start

実験を最短手順で実行するための基本的なコマンドフローです。

**ステップ1. 前処理とCVデータセットの作成**
まず、生データから前処理・正規化を行い、6-fold交差検証用の学習データセットを構築します。
```bash
python scripts/run_manual_cv.py
```

**ステップ2. 学習の実行**
構築されたデータセットを用いて、深層学習モデルを学習させます。
以下は、片足のセンサデータ（入力）から床反力（GRF）のみをCNNモデルで推定評価する実行例です。
```bash
PYTHONPATH="$(pwd)" ./env/bin/python scripts/train_cv.py \
    --exp_name cnn_grf_single \
    --model_type cnn \
    --input_type single_leg \
    --target_type grf_only
```

**ステップ3. 結果の確認**
学習が完了すると、自動的に `outputs/experiments/` 以下に実験パラメータや学習曲線を含む結果一式がディレクトリとして保存されます。平均R2スコアなどの評価は出力先の `summary.csv` で確認してください。

## 2. プロジェクト概要

### 研究の目的と背景
通常の歩行解析には、赤外線カメラでマーカーを追従する「モーションキャプチャシステム」と、地面に埋め込まれた「フォースプレート（床反力計）」という高価かつ専用の実験環境が必要です。

本研究は、**「靴の中敷き（足底圧力センサ）」と「身体に装着する小型センサ（IMU：加速度・ジャイロセンサ）」というウェアラブルセンサの計測データ（入力）のみから、深層学習による時系列回帰モデルを用いて、モーションキャプチャ相当の「下肢関節角度」および「床反力（GRF）」情報（教師信号）を高精度に予測・推定すること**を目的としています。

これにより、高額な設備がない屋外環境や臨床現場など、あらゆる環境における精密な歩行解析を実現するという重要な学術的・実践的意義を持っています。

## 4. データの流れ

「生データ」が計算機に入力され、「AIの学習結果」として出力されるまでの全体のデータフローです。

```text
【1. データの準備】
  [raw data] (足底圧力、IMU、モーションキャプチャ)
      ↓
【2. 前処理（ノイズ除去と同期）】
  [src/preprocessing モジュール群] 
      ↓
【3. 歩行周期の分割】
  [stride抽出] (1歩単位で波形を切り出し、0~100%の長さに正規化)

【4. 均等化】
  [正規化処理] (身長・体重の違いによる影響の補正)
      ↓
【5. データセットの分割】
  [manual 6-fold CV 作成] (未知の被験者に対する汎化性能を検証するための分割)
      ↓
【6. 学習用ファイルの生成】
  [train.pkl / val.pkl / test.pkl] 
      ↓ （ここからモデルの学習・評価プロセス）
【7. 特徴量の動的抽出】
  [FeatureSelector] (実験条件に応じ、入力センサ種別や予測対象を動的に切り出し)
      ↓
【8. PyTorch データ読み込み】
  [PyTorch Dataset / DataLoader] (深層学習用のバッチデータを作成)
      ↓
【9. 学習と誤差最適化】
  [モデル学習 (engine.py)] (ネットワークが予測し、教師信号との誤差を修正、パラメータ更新)
      ↓
【10. 未知のデータでの評価】
  [テストセット評価] (学習モデルに一切含まれていない被験者のデータで最終性能テスト)
      ↓
【11. 実験完了】
  [結果保存] (最適化された重みや summary.csv に結果を出力)
```

## 5. 前処理パイプライン詳細 (`src/preprocessing/`)

生データに存在するノイズやサンプリング周波数の違いを補正し、学習モデルが処理可能なテンソル形式へ変換するモジュール群です。

- **`config.py`**
  - **役割**: 各被験者の体重、各デバイスのサンプリング周波数、対応するマーカー名などの実験固有の定数情報。
  - **接続**: 各処理モジュールから設定値として呼び出されます。
- **`loader.py`**
  - **役割**: 各被験者の計測CSVデータをPandas DataFrameとしてメモリ上に一括ロードします。
  - **接続**: `cleaner.py` へデータを渡します。
- **`cleaner.py`**
  - **役割**: 解析に不要な列の削除、および列名の統一処理を実行します。
  - **接続**: `resample.py` へ渡します。
- **`resample.py`**
  - **役割**: 解析デバイス間（モーションキャプチャとIMUなど）におけるサンプリングレートの違いに対して線形補間を行い、時間を等間隔に統一します。
- **`filter.py`**
  - **役割**: バターワースフィルタ等を用いて、高周波成分等の観測ノイズを平滑化し、モデルの学習の阻害要因を排除します。
- **`kinematics.py`**
  - **役割**: 3次元空間上の各マーカー位置座標データから、股関節・膝関節・足関節などの各関節角度を算出します（回帰タスクの教師信号生成）。
- **`sync.py`**
  - **役割**: 各センサの計測開始時刻の微小な誤差（ミリ秒単位）を、相互相関関数を用いて算出・補正し、完全に時刻を同期させます。
- **`stride.py`**
  - **役割**: ヒールストライク（踵の接地）を波形の特徴から検出し、歩行周期における「1歩」分を抽出し、時間軸を100%に正規化します。
- **`normalization.py`**
  - **役割**: 身長や体重など被験者ごとの体格差に起因するスケールの差異をなくすため、正規化または標準化を行います。
  - **重要**: データリーク防止のため、標準化パラメータ（Mean、Variance）は必ず**訓練データ（Train）のみ対象に計算**し、検証・テストデータに適用します。
- **`feature_selector.py`** 
  - **役割**: （詳細後述）結合された52列の多次元配列から、実験条件に合わせて特定のセンサ情報および予測対象列を動的にスライス（抽出）します。

★ **`pipeline.py`** (統括モジュール)
上記の全モジュールを順次呼び出し、「データロード〜分割・正規化」までの前処理工程を完全自動で取りまとめるためのメインスクリプトです。

## 6. manual 6-fold CV の仕組み

今回の被験者は合計12名であり、汎化性能を正確に評価するため、被験者単位での交差検証モデル（Group CV）を採用しています。

**【CVの設定】**
- **被験者単位（Group）での分割**
- **各Foldにおける割当**: Train（訓練）: 8名 / Validation（検証）: 2名 / Test（最終評価）: 2名
- **全6 Fold** の実行を通して、12名全員がいずれかのFoldにおいて必ず Test データを1度担当するように分割リストを手動で固定しています。

**【なぜこの設計なのか（被験者リークの防止）】**
人間の歩行には個々の被験者ごとに特有の強い癖（個性）が存在します。もし、特定の被験者のデータが訓練データとテストデータの両方に含まれた場合、モデルは物理的な歩行の一般的法則を獲得するのではなく、単に「対象の被験者の癖を変数として記憶する」だけとなってしまいます。
これをデータリーク（Data Leakage）と呼びます。モデルの真の汎化性能（全くの未知の人間に対する予測精度）を計測するためには、被験者単位での厳格なデータの分離分割が不可欠となります。

**`scripts/run_manual_cv.py` の役割**
定義した6 Foldの構成リストに基づき、データ漏洩を引き起こすことなく、各Fold専用の `train.pkl`, `val.pkl`, `test.pkl` を作成し `data/processed/cv/` に出力するための生成スクリプトです。

## 8. FeatureSelector の役割

**なぜ必要なのか：**
「足底圧力情報のみを入力とした場合の推論精度」や「両脚の情報を活用した際の精度向上率」など、**機能ごとにデータセットを作成し直す手間を削減し、様々な入力設定におけるアブレーション・スタディを即座に実行可能とするため**です。
`src/preprocessing/feature_selector.py` は、入力された52次元の多変数アレイから、実行設定に応じて柔軟な抽出を行います。

**【活用可能な入力 (X)】**
- `single_leg`: 同側の14ch（Pressure+IMU）を入力
- `bilateral`: 両側の28ch（同側+対側）を入力
- `pressure_only` (single/bilat): 足底圧力情報のみを入力
- `imu_only` (single/bilat): IMU信号のみを入力

**【活用可能な予測データ / 教師信号 (y)】**
- `all`: 関節角度(9ch) + 床反力(3ch) = 12ch すべてを教師信号とする
- `angles_only`: 関節角度(9ch) のみを教師信号とする
- `grf_only`: 床反力(3ch) のみを教師信号とする

## 9. 学習インフラストラクチャ詳細

分析データをPyTorchで柔軟かつスケーラブルに学習させるための基盤環境です。

### `src/models/` (モデルネットワーク定義)
- **`cnn.py` (CNN)**: 時間軸方向に一次元畳み込みフィルタを適用し、ヒールストライク時等の局所的な波形パターン特徴を効率的に抽出するモデルです。
- **`bilstm.py` (BiLSTM)**: 歩行動態の連続的な時間依存関係（文脈）において、「過去から未来」と「未来から過去」の両視点から現在の連続状態を滑らかに推論するRNNベースのモデルです。
- **`transformer.py` (Transformer)**: セルフアテンション機構を用いて長距離の時間依存関係を全体俯瞰的に捉えるモデルですが、ハイパーパラメータの調整感度が比較的高い特徴があります。
- **`__init__.py`**: 外部モジュールから設定文字列 (`'cnn'`, `'bilstm'` 等) により、対象モデルのインスタンスを動的に生成するファクトリ関数の役割を担います。

### `src/training/` (学習最適化と評価)
- **`dataset.py`**: `.pkl` データをパースし、`FeatureSelector`を介して動的に `X` と `y` を抽出して バッチ作成用のPyTorch `Dataset` オブジェクトを構築します。
- **`loss.py`**: 様々な最適化基準のため、`MSELoss` や、特定の予測変数（例: 床反力の垂直成分など）に対して強調的勾配・損失を与える `WeightedMSELoss` を実装しています。
- **`engine.py`**: 損失演算およびパラメータ更新を行う学習ループや、過学習防止のための Early Stopping を制御する `Trainer` 制御クラスです。
- **`metrics.py`**: モデルの性能評価指標として、RMSE、NRMSE、MAE、R2スコアなどの回帰指標を算出する関数群です。
- **`config.py`**: `train_cv.py` 実行時に受け取る `argparse` の引数とそのデフォルト設定を定義するモジュールです。

### `src/` 配下のルート等
- **`src/utils.py`**: ランダム成分起因による実験間の精度の変動を無くすために乱数シードを固定化する `seed_everything` などの共通関数を定義しています。

## 10. `train_cv.py` の実行フロー

`scripts/train_cv.py` をコマンドラインから実行した際、内部では以下の順序でプロセスが進行します。

1. **Configの読み込み**: 引数として設定されたパラメータのパースと決定。
2. **Seed固定**: `seed_everything()` による実験再現性の担保。
3. **ループ実行**: Fold 1 の学習・検証プロセスを開始。
4. **Dataset構築**: 当該Foldの `train.pkl` 等を読み込み、バッチ分割用の `DataLoader` をインスタンス化。
5. **Model生成**: 指定されたモデルや次元情報を元に推論ネットワークを設計。対応する損失関数とOptimizerも構成。
6. **Train loopの開始**: `Trainer` が学習データに対してミニバッチ学習とパラメータ最適化を実行。
7. **Best Model保存**: Validation Lossを指標に最良性能時のネットワーク重みを `best_model_fold1.pth` として保存。
8. **Testデータの評価**: 未知のTest被験者データに対して順伝播させ、各Metrics（R2, RMSEなど）による絶対性能評価を実施。
9. **Foldの進行**: Fold 2 ~ Fold 6 に対して手順3〜8の工程を繰り返し適用。
10. **総合Summary保存**: 全6Foldのテスト成績を平均および標準偏差として集約し、`summary.csv` として記録・クローズ。

## 11. 実行方法の具体例

設定パラメータを変更することで多様な比較実験を1つのコマンドで再現できます。

**例1：CNNを活用して片足の全センサデータから床反力(GRF)のみ回帰推論**
```bash
PYTHONPATH="$(pwd)" ./env/bin/python scripts/train_cv.py \
    --exp_name cnn_grf_single \
    --model_type cnn \
    --input_type single_leg \
    --target_type grf_only
```

**例2：両脚の全データを入力とし、全12変数をTransformerで予測（重み付きLossを適用）**
```bash
PYTHONPATH="$(pwd)" ./env/bin/python scripts/train_cv.py \
    --exp_name transformer_all_bilateral \
    --model_type transformer \
    --input_type bilateral \
    --target_type all \
    --use_weighted_loss \
    --epochs 300
```

**例3：足底圧力情報のみを入力し、BiLSTMで関節角度を推測**
```bash
PYTHONPATH="$(pwd)" ./env/bin/python scripts/train_cv.py \
    --exp_name bilstm_pressure2angles \
    --model_type bilstm \
    --input_type pressure_bilateral \
    --target_type angles_only \
    --lr 0.0005
```

## 12. 出力結果ディレクトリの見方

学習が完了すると、自動的に `outputs/experiments/【設定名_日時】/` 階層内に結果が保存されます。

**outputs ディレクトリの保存構造例**:
```text
outputs/experiments/cnn_grf_single_20260416_160500/
├── config.json
├── best_model_fold1.pth
├── best_model_fold2.pth
├── ...
├── learning_curve_fold1.png
├── learning_curve_fold2.png
├── ...
└── summary.csv
```

- **`config.json`**: 引数で指定されたパラメータのダンプデータ。事後分析における実験条件のトレーサビリティを担保します。
- **`best_model_fold{X}.pth`**: `Transformer` などのPyTorchモデルが各FoldでValidation Lossを最小化した際（最も汎化性能が高まった状態）の重みパラメータです。将来の未知データにおける推論タスクに使用します。
- **`learning_curve_fold{X}.png`**: ネットワーク学習過程における Train Loss と Val Loss の推移グラフです。右肩下がりかつ乖離が少なければ収束しており、Val Lossが途中から上昇に転じていれば過学習の目安となります。
- **`summary.csv`**: 全6 Fold毎の推論誤差や決定係数（R2）と、最下段に集計された **Mean（平均）** および **Std（標準偏差）** を記載した最終検証結果リストです。各アーキテクチャの性能比較には主にこのMean指標を用います。

## 13. 今後の拡張ポイント

本プロジェクトにおけるベースインフラストラクチャーは、今後の柔軟な機能拡張を志向して疎結合に設計されています。

- **新規モデル・アーキテクチャの追加**: `src/models/` 内に新しい構造（TabNetやハイブリッドモデル等）を定義し、`__init__.py` でインスタンス化を追加するだけで既存コードから透過的に利用可能となります。
- **Optuna等による自動調整化**: `train_cv.py` におけるハイパーパラメータの引数を Optuna トライアル変数に連結することで、探索範囲を容易に自動化できます。
- **推論および可視化スクリプトの開発**: 保存済みの `best_model.pth` をロードし、外部実地データに対してリアルタイムに近い速度で推論および物理的可視化（アニメーションGIF化等）を行うツールを開発予定です。
- **新しい入力特徴量の拡張**: モーションキャプチャ等から追加した「時間・距離空間的パラメータ」等に対しても、`feature_selector.py` 内に抽出論理を追加することで対応可能です。
